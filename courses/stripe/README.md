# Stripe API & Product Knowledge

Deep study of Stripe's real platform, API objects, and integration patterns. Designed for someone preparing for a technical screening — not a coding drill, but fluency with the actual product.

## Lessons

| # | File | Topic |
|---|------|-------|
| 01 | `01-api-fundamentals-and-core-objects.md` | API design, authentication, core objects, idempotency, metadata, webhooks |
| 02 | `02-payments-deep-dive.md` | PaymentIntents lifecycle, PaymentMethods, SetupIntents, Checkout, refunds, disputes |
| 03 | `03-billing-and-subscriptions.md` | Products, Prices, Subscriptions lifecycle, Invoices, proration, metered billing, Billing Portal |
| 04 | `04-connect-and-platform-payments.md` | Account types, charge models, onboarding, application fees, transfers, payouts |

## How to use

Each lesson is structured as:
- **Concepts** — how the real Stripe objects work, their statuses, and relationships
- **API patterns** — real API calls and parameters you'd use
- **Decision frameworks** — when to use what and why
- **Check yourself** — questions to verify understanding (answers inline)

Read sequentially. Lessons 01-02 are the highest priority for a general technical screening.
